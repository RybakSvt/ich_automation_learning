# Homework 6: UI Automation Tests with Selenium - Page Object Model (POM)

## Описание задания
Автоматизация тестирования процесса покупки в интернет-магазине с использованием паттерна Page Object Model (POM) для организации кода и улучшения поддерживаемости тестов.

## Файлы в папке hw6

### Page Objects:
- `login_page.py` — работа со страницей авторизации
- `inventory_page.py` — работа со страницей товаров
- `cart_page.py` — работа с корзиной
- `checkout_page.py` — работа с оформлением заказа

### Тесты и настройки:
- `test_checkout.py` — основной тест процесса покупки
- `conftest.py` — настройки браузера и фикстуры pytest
- `requirements.txt` — зависимости проекта
- `README.md` — документация

## Как запустить тесты

```bash
# Запуск всех тестов
python -m pytest

# Запуск с подробным выводом
python -m pytest -v

# Запуск конкретного теста с выводом print-ов
python -m pytest tests/test_checkout.py -v -s

# Запуск из корневой папки hw6
cd hw6
python -m pytest tests/test_checkout.py -v

Особенности реализации
Архитектура Page Object Model:
LoginPage - управление авторизацией

InventoryPage - работа с товарами и корзиной

CartPage - управление корзиной покупок

CheckoutPage - оформление заказа и проверка суммы

Ключевые особенности:
Использование WebDriverWait для явных ожиданий элементов

CSS селекторы для надежных локаторов элементов

Инкапсуляция логики работы со страницами в отдельных классах

Четкое разделение тестовой логики и работы с DOM

Фикстуры pytest для управления жизненным циклом браузера

Преимущества подхода

Поддерживаемость - изменения на странице требуют правок только в одном месте

Переиспользуемость - Page Objects могут использоваться в разных тестах

Читаемость - тесты отражают бизнес-логику, а не технические детали

Надежность - использование явных ожиданий и стабильных локаторов

Масштабируемость - легко добавлять новые тесты и страницы
