# Homework 7: API Automation Tests for Employee Management

## Описание задания
Автоматизация тестирования API для управления сотрудниками с созданием вспомогательных методов и тестов для проверки корректности работы.

## Файлы в папке hw7
`employee_api.py` — класс с методами для работы с API сотрудников

`test_employee.py` — тесты для проверки API

`README.md` — это описание

## Как запустить тесты

# Запуск всех тестов
python test_employee.py

# Запуск с подробным выводом
python -m pytest test_employee.py -v

# Запуск конкретного файла
python -m pytest test_employee.py::test_create_employee -v
python -m pytest test_employee.py::test_get_employee -v
python -m pytest test_employee.py::test_update_employee -v

## Цель задания
Разработать автоматические тесты, которые проверяют корректность работы API для управления сотрудниками. Создать класс EmployeeApi для создания вспомогательных методов.

## Функциональность тестов

## test_employee.py:

**test_create_employee():**
- Создание сотрудника со всеми полями
- Проверка сохранения всех данных в базе
- Верификация корректности сохраненных полей

**test_get_employee():**
- Получение информации о сотруднике по ID
- Проверка наличия основных полей в ответе
- Вывод информации о найденном сотруднике

**test_update_employee():**
- Обновление данных сотрудника через PATCH
- Использование токена авторизации для запроса
- Проверка обновленных полей (фамилия, email, телефон, статус)

**test_check_patch_fields():**
- Проверка, какие поля можно обновить через PATCH
- Тестирование обновления разных полей
- Определение обновляемых и необновляемых полей

## Особенности реализации
Использование токена авторизации в PATCH-методе

Уникальные данные для каждого теста во избежание конфликтов

Проверка всех полей сотрудника при создании

Обработка ошибок и исключений

Минимизация дублирования кода
